## 🔥 КРИТИЧНО — Краш при старте

**Причина:** `instance` в `LoParkour.java` объявлен но никогда не присваивается.
`Config` enum при статической инициализации зовёт `LoParkour.getInFolder()` → NPE → краш.

**Фикс:**
```java
@Override
public void onEnable() {
    instance = this;   // ← этого нет
    super.onEnable();  // вызывает enable()
}
```

- [ ] Добавить `instance = this` перед `super.onEnable()`
- [ ] Пересобрать: `./gradlew shadowJar`
- [ ] Проверить что сервер стартует без ошибок

## 🐛 БАГИ КОНКУРЕНТОВ — Устранить у нас лучше

Эти проблемы убили Infinite Parkour в глазах сообщества. У нас их быть не должно.

### Ghost blocks — блоки остаются висеть после сессии
У IP это главная жалоба — после дисконнекта или краша блоки паркура остаются в мире.

- [ ] Аудит `ParkourGenerator.disable()` и `World.delete()` — всё ли чистится
- [ ] Список всех размещённых блоков писать в сессию, чистить при любом выходе
- [ ] Обработать `PlayerQuitEvent`, `PlayerKickEvent`, `ServerShutdownEvent`
- [ ] Тест: войти → выйти через Alt+F4 → зайти снова → блоков нет

### World generation баг — генерируется лишний overworld
У IP при определённых условиях создавался новый мир и жрал память.

- [ ] Убедиться что `World.create()` не создаёт новый мир если уже есть
- [ ] Проверить что мир регистрируется в `worlds` конфиге и не пересоздаётся
- [ ] Логировать создание/удаление мира

### Scoreboard баг
У IP scoreboard ломался и фиксился только отключением.

- [ ] Использовать LoLib Sidebar API вместо ручного scoreboard
- [ ] Обернуть в try/catch, при ошибке — пересоздать sidebar, не ронять плагин

### Лидерборды из коробки без плагинов
У IP нужен HolographicDisplays, документация невнятная, люди не разбирались.

- [ ] Лидерборды через встроенный LoLib holograms (пакеты, без зависимостей)
- [ ] `HoloHook` только как опциональное улучшение
- [ ] Авто-создание голограммы командой `/lp leaderboard create <mode> <location>`
- [ ] Документация прямо в `config.yml` комментариями

### Inventory recovery
У IP терялся инвентарь при краше. Было, но работало криво.

- [ ] Команда `/lp recoverinventory` — восстановить инвентарь из сохранённого снапшота
- [ ] Снапшот сохранять async на диск при входе в паркур
- [ ] Восстанавливать автоматически при старте если игрок был в паркуре

## 🎮 РЕЖИМЫ — Доработка существующих

### ElytraMode → полноценный Elytra Parkour
Сейчас: выдаётся элитра + ракеты, летишь над обычными блоками. Это не режим.
Нужно: 3 подрежима по аналогии с Infinite Elytra Parkour (6 режимов).

**Подрежим 1 — Classic (бесконечный маршрут)**
- [ ] `ElytraGenerator` — генерирует широкие платформы и коридоры для полёта
- [ ] Firefly буст при подборе ракеты в мире
- [ ] Счёт за дистанцию

**Подрежим 2 — Precision (точность через кольца)**
- [ ] Кольца из стекла/стеклопанелей — процедурная генерация
- [ ] Кольца разного диаметра (большие → маленькие с прогрессом)
- [ ] Счёт за точность: пролетел по центру → 3 очка, по краю → 1 очко, пропустил → 0
- [ ] Хитбокс кольца через регионы LoLib (Cuboid/Cylinder)

**Подрежим 3 — Obstacle (уклонение)**
- [ ] Стены с дырами, колонны, горизонтальные балки
- [ ] Нарастающая скорость через эффект скорости
- [ ] Жизни (3 касания = конец)

**Ветровые зоны (просили пользователи Elytra плагина)**
- [ ] Зоны ускорения (tailwind) — зелёные частицы, +скорость
- [ ] Зоны замедления (headwind) — красные частицы, -скорость
- [ ] Боковой ветер — толчок в сторону

**Общее для ElytraMode:**
- [ ] Конфиг: стартовые ракеты, размер колец, скорость нарастания
- [ ] Scoreboard: текущий подрежим, очки, ракеты
- [ ] Нет стандартного `ParkourGenerator` — только свой `ElytraGenerator`

### SpeedrunMode
- [ ] Текущий рекорд в scoreboard в реальном времени
- [ ] Сплиты: +δ время на 10/25/50/100 прыжках (зелёный если быстрее рекорда, красный если медленнее)
- [ ] Ghost replay при побитии рекорда

### HardcoreMode
- [ ] Настройка количества жизней (1–5) в конфиге
- [ ] Индикация жизней в scoreboard (сердца)
- [ ] Опция: потеря наград при потере жизни vs только при нуле жизней

### GravityShiftMode
- [ ] Предупреждение за 3 прыжка до смены (title на экране)
- [ ] Визуальный эффект смены (частицы + звук)
- [ ] Scoreboard показывает следующий эффект

## 🆕 НОВЫЕ РЕЖИМЫ

### LavaRush (просили пользователи IP)
Паркур с поднимающейся лавой снизу.

- [ ] `LavaRushMode extends DefaultMode`
- [ ] Лава-блок (или FakeBlock) поднимается каждые N тиков
- [ ] Скорость подъёма нарастает с прогрессом
- [ ] Счёт = высота от начального уровня
- [ ] Конфиг: начальная скорость, нарастание, визуальный тип (настоящая лава / красные частицы)
- [ ] Совет: FakeBlock через пакеты (LoLib packets) — не грузит сервер

### LobbyParkour (Infinite Parkour Plus фича)
Фоновый паркур в зоне ожидания сервера. Игрок просто прыгает пока ждёт.

- [ ] Отдельный режим без очков и лидерборда
- [ ] Не сохраняет статистику
- [ ] Настройка зоны в конфиге (мир + координаты)
- [ ] Авто-старт при входе в зону

### SessionLobby — меню активных сессий (IP Plus фича)
Меню где видно кто сейчас играет, можно присоединиться или зайти зрителем.

- [ ] `SessionMenu extends BaseMenu` — список всех активных сессий
- [ ] Каждая сессия: режим, игроки, счёт, кнопки "Присоединиться" / "Смотреть"
- [ ] Обновление в реальном времени (каждые 2 секунды)
- [ ] Команда `/lp sessions`

### NatureParkour (Infiniti Parkus стиль)
Расслабленный паркур по природной местности. Без таймера.

- [ ] `NatureGenerator`:
  - Блоки: трава, земля, дерево, листья, цветы, камень, мох
  - Широкие платформы (2–4 блока), мостики из дерева, арки из листьев
  - Биомные темы: лес / горы / пляж / снег (переключаются в настройках)
- [ ] Нет таймера — счёт за пройденные блоки
- [ ] Коллаборативный: 2–4 игрока вместе на одном паркуре

### ZigZagMode
- [ ] Строго зигзаг (лево/право/лево/право) с нарастающей скоростью
- [ ] Только одиночные блоки, без схематик
- [ ] Тренировочный режим для стрейфов

### DuelsMode (мультиплеер)
- [ ] 2 игрока, параллельные трассы, 60 секунд
- [ ] Кто больше прыжков — победил
- [ ] Саботаж каждые 20 прыжков: выбрать дебафф противнику (замедление / слепота / левитация)
- [ ] Очередь матчей
